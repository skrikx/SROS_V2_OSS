# Contributing

## Foundation Rule

SROS V2 uses a constitution-bound, workflow-first rebuild model.
Contributors must preserve local-only CLI law and avoid introducing parallel architecture.

## During W01

Only scaffold and bootstrap changes are allowed. Do not add subsystem behavior.

## General Expectations

- Keep changes inside the active workflow scope.
- Prefer minimal, compile-safe increments.
- Do not introduce daemon/API/UI enterprise surfaces.
- Preserve canonical chain and governance constraints.
