# Contributing

SROS V2 should feel worth extending.

## Before you open a PR

Run:

- `go test ./...`
- `go run ./cmd/sros verify`
- `go run ./cmd/sros examples run`
- `go run ./cmd/sros test smoke`

## Contribution gravity rules

- preserve local-only CLI law
- preserve distinct run, trace, memory, policy, evidence, and release truths
- keep docs proof-first and truthful
- prefer real examples and real artifacts over abstract explanation
- do not widen into hosted or enterprise shell work

## Strong places to contribute

- docs and examples that expose real machine value
- CLI inspectability and operator ergonomics
- replay, receipt, and release-pack quality
- regression and smoke coverage
